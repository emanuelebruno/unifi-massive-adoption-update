# README rule
Ogni modifica funzionale, strutturale, architetturale o di deploy deve aggiornare `README.md`.
Aggiornare almeno:
- panoramica
- stato attuale
- architettura
- funzioni implementate
- endpoint/API
- avvio/test/deploy
- storico modifiche funzionali
Non usare il README per micro-refactor invisibili all’uso. Ogni nuova route, schermata, modello, dipendenza importante o cambio proxy/deploy va documentato.
